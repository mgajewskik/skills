# Compass and Core Model

Primary sources:
- https://diataxis.fr/start-here/
- https://diataxis.fr/compass/
- https://diataxis.fr/map/

## The core idea

Diátaxis says documentation serves **four distinct user needs**, so it should take **four distinct forms**:

- tutorials
- how-to guides
- reference
- explanation

The forms are not arbitrary categories. They come from two dimensions:
- **action vs cognition**
- **study vs work**

In the deeper theory, these correspond to **acquisition vs application** of skill.

## The compass

Ask only two questions:

1. Does the content primarily **inform action** or **inform cognition**?
2. Does it primarily serve the reader’s **study** or **work**?

| If the content… | …and serves the reader’s… | …then it belongs to… |
| --- | --- | --- |
| informs action | study | tutorial |
| informs action | work | how-to guide |
| informs cognition | work | reference |
| informs cognition | study | explanation |

Use the compass:
- when deciding what to write
- when deciding what existing content already is
- when the page feels wrong but the problem is not obvious
- at document, section, paragraph, or sentence level

## Quick profiles

### Tutorial
- question answered: **Can you teach me to…?**
- purpose: provide a learning experience
- shape: a lesson with a managed path

### How-to guide
- question answered: **How do I…?**
- purpose: help an already-competent user achieve a goal
- shape: practical directions through a real task or problem

### Reference
- question answered: **What is…?**
- purpose: describe the machinery accurately
- shape: neutral, structured information to consult

### Explanation
- question answered: **Why…?** or **Can you tell me about…?**
- purpose: deepen understanding
- shape: discussion, context, rationale, perspective

## Rules that prevent misclassification

- The **same topic** can appear in all four quadrants.
- Complexity does **not** decide the quadrant.
- Beginner vs advanced does **not** decide the quadrant.
- A document can be mixed. When that happens, fix the mixture instead of forcing one label onto everything.
- The map is a guide to relationships, not a command to create four empty site sections.

## Fast signals

Likely tutorial signals:
- guided sequence in a controlled setting
- explicit reassurance and expected results
- learner safety and success are the writer’s responsibility

Likely how-to signals:
- a concrete real-world goal
- conditional branches and judgment calls
- assumes reader competence

Likely reference signals:
- tables, lists, options, fields, APIs, commands, warnings
- neutral description with standard patterns
- meant to be consulted while working

Likely explanation signals:
- history, rationale, tradeoffs, context, alternatives
- answers why something is the way it is
- meant to be read away from the immediate task

## When in doubt

- If explanation appears inside action-oriented content, move or link it out.
- If steps appear inside cognition-oriented content, extract them into a tutorial or how-to guide.
- If the user is **studying**, bias toward tutorial or explanation.
- If the user is **working**, bias toward how-to guide or reference.
